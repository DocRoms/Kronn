- **ID**: TD-20260721-hermetic-notify-e2e
- **Area**: Workflow engine / E2E / CI
- **Problem (fact)**: `workflow-linear-3-step.spec.ts` exercises the real Notify dispatch through two public `httpbin.org` endpoints. An up-front probe cannot close the probe-to-run race: CI run `29819560314` completed the first Notify, then the second request reached the executor's 30-second timeout and failed the E2E job. The narrow flake guard prevents that known transport/5xx case from creating a false red, but an externally unavailable run is skipped and therefore supplies no Notify E2E signal. The executor already has a local-server policy used by Rust tests, but the production runner always selects the enforcing path. Making the Playwright backend accept localhost just for convenience would weaken or complicate a security invariant and is out of scope for the 0.8.13 CI repair. - **Impact**: CI can spend up to 30 seconds on an external failure and skip the only browser-level runner-to-Notify scenario. Repeated public-network failures remain operational noise even when they no longer fail the build.
- **Where (pointers)**: `frontend/e2e/specs/workflow-linear-3-step.spec.ts`, `backend/src/workflows/runner.rs`, `backend/src/workflows/notify_step.rs`, and `.github/workflows/ci-test.yml`.
- **Suggested direction (non-binding)**: Add a hermetic Rust integration harness that executes the full workflow runner against a local webhook with an injected test-only Notify policy; do not add a runtime production bypass for private targets. Once runner dispatch, interpolation and state propagation are covered there, replace the browser scenario's public Notify calls with deterministic zero-network steps (for example a JsonData/Gate resume chain), keeping browser coverage focused on API lifecycle and persisted run state.
- **Acceptance criteria**: The full runner-to-Notify path is tested against a process-local HTTP server; the test proves interpolated request data and persisted step results; Playwright workflow tests make no public-network request; the production SSRF policy remains unchanged and its private-target regression test stays green; CI has no skip path for the linear workflow scenario.
- **Next step**: design the injected test policy at the runner boundary and prove that it is unreachable from production builds before changing the Playwright scenario.
